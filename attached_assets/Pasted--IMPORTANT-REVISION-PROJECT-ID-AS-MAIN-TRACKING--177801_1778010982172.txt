==================================================
IMPORTANT REVISION: PROJECT ID AS MAIN TRACKING REFERENCE
==================================================

Think carefully before changing the system.

Do not add anything extra that I am not asking.

The rule is:

CRM Module and Sales Module can identify and search by:
- Client Code
- User Unique ID
- Lead ID
- Client Name
- Mobile
- Email
- LPO Number
- Quotation Number
- Project ID

But all other modules must mainly identify and work by PROJECT ID only.

This applies to:
- Accounts Module
- Procurement Module
- Inventory Module
- Project Module
- Delivery Module
- Documents / PDF Export
- Reports related to project cost, project billing, project delivery, project procurement, and project inventory

The purpose:
After Sales receives/registers/imports the client LPO, the system creates one unique PROJECT ID.
After that, all operational departments must follow that PROJECT ID as the main reference.

==================================================
1. DO NOT CHANGE CRM AND SALES LOGIC
==================================================

Do not remove existing CRM and Sales tracking.

CRM and Sales should still show:
- Client Code
- User Unique ID
- Lead ID
- Client Name
- Mobile
- Email
- Sales User
- LPO Number
- Quotation Number
- Project ID

CRM and Sales are the front-office modules.

They manage:
- Client inquiry
- Lead tracking
- Sales user tracking
- Quotation
- LPO registration
- Project ID creation

Important:
Only Sales/LPO registration should create the Project ID.

==================================================
2. PROJECT ID CREATION LOGIC
==================================================

When Sales User registers LPO details or uploads/imports the client LPO document, the system must create a unique Project ID.

Project ID format:
PRJ-YYYY-000001

Rules:
- Project ID must be created only once per approved/verified LPO.
- If LPO already has Project ID, do not create another Project ID.
- Project ID must remain permanent.
- Project ID must connect to Client, Lead, Quotation, LPO, Sales User, and Product/Service.
- Project ID must be used by all other modules after creation.

==================================================
3. ACCOUNTS MODULE — PROJECT ID FOCUS ONLY
==================================================

In Accounts Module, the main tracking field must be PROJECT ID.

Accounts users should mainly search, filter, create, and review records by Project ID.

Accounts documents connected to Project ID:
- Proforma Invoice
- Tax Invoice
- Payment Received
- Payment Made
- Expenses
- Journal Entries
- VAT Report where applicable
- Client Statement where applicable
- Project Balance Report

Accounts screens must show Project ID clearly.

Accounts search bar placeholder:
“Search by Project ID or Document Number”

Accounts filters:
- Project ID
- Document Type
- Date Range
- Payment Status
- Invoice Status

Do not make User Unique ID the main working reference inside Accounts.
Do not make Client Code the main working reference inside Accounts.
Client Code can exist in document details, but Accounts user’s main focus must be PROJECT ID.

Accounts dashboard cards:
- Project invoices
- Project payments
- Project balance
- Project expenses
- Project receivables
- Project profitability

==================================================
4. PROCUREMENT MODULE — PROJECT ID FOCUS ONLY
==================================================

In Procurement Module, the main tracking field must be PROJECT ID.

Procurement users should create and manage all procurement activities under Project ID.

Procurement documents connected to Project ID:
- Purchase Request
- RFQ
- Supplier Quotation
- Purchase Order
- Supplier Bill if available
- Material Request
- Procurement Comparison

Procurement search bar placeholder:
“Search by Project ID, PR Number, RFQ Number, or PO Number”

Procurement filters:
- Project ID
- Supplier
- Purchase Status
- Approval Status
- Date Range

Procurement workflow:
1. User selects Project ID.
2. System loads project details.
3. User creates Purchase Request or RFQ under that Project ID.
4. Purchase Order remains linked to the same Project ID.
5. Procurement cost updates project cost summary.

Do not make Client Code the main field in Procurement.
Do not make User Unique ID the main field in Procurement.
Project ID is the main operational reference.
